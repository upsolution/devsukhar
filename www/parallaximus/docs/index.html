Parallaximus Documentation

Step-by-step guide on how to create your widget

1. Upload parallaximus folder to your server and include the required files in your page (usually in the head section):

Basic parallaximus styles should be included just before your page styles.

<!-- Basic Parallaximus styles  -->
<!-- (should be inserted before your page styles) -->
<link rel="stylesheet" type="text/css" href="parallaximus/parallaximus.css" media="all">

<!-- (!) Your page styles are here. -->

<!-- Parallaximus requires JQuery 1.7+ -->
<!-- Note: if you already have jQuery at this moment, you shouldn't include it once more. -->
<script type="text/javascript" src="parallaximus/jquery-1.9.1.min.js"></script>

<!-- Parallaximus script (should be placed after Parallaximus styles definition) -->
<script type="text/javascript" src="parallaximus/jquery.parallaximus.min.js"></script>
<!-- Basic Parallaximus styles  -->
<!-- (should be inserted before your page styles) -->
<link rel="stylesheet" type="text/css" href="parallaximus/parallaximus.css" media="all">

<!-- (!) Your page styles are here. -->

<!-- Parallaximus requires MooTools 1.4+ -->
<!-- Note: if you already have MooTools at this moment, you shouldn't include it once more. -->
<script type="text/javascript" src="parallaximus/mootools-core-1.4.5-full-nocompat-yc.js"></script>

<!-- Parallaximus script (should be placed after Parallaximus styles definition) -->
<script type="text/javascript" src="parallaximus/mootools.parallaximus.min.js"></script>

2. Choose the most suitable demo (from demos list) for your case and:

  • open its PSD (archive: psd/{DemoName}.psd) to create your own widget appearance;
  • export layers as trimmed images (as done in demo example) and upload them to your server image location;
  • copy demo HTML (from archive: demos/{DemoName}/index.html file) to your page body section (to the place where your widget should be located):

    Below is an approximate example of HTML code, specific demo HTML will differ a bit.

    <div class="w-parallaximus width_fixed height_fixed">
	<div class="w-parallaximus-layer index_1">
		<img src="image.png" width="165" height="219" alt="">
	</div>
	<div class="w-parallaximus-layer index_2">
		<img src="image.png" width="130" height="42" alt="">
	</div>
</div>
  • enter correct sources, widths and heights of images;

    The first layer is the farthest from the viewpoint. The last one is the nearest.

  • copy demo styles (from archive demos/{DemoName}/index.html file) to your page styles CSS file:

    Below is an approximate example of CSS code, specific demo CSS will differ a bit.

    .w-parallaximus {
	width: 240px;
	height: 400px;
	/* If you widget has background place it here */
	background: url(background.jpg);
	}
	.w-parallaximus-layer.index_1 {
		width: 400px;
		height: 500px;
		}
		.w-parallaximus-layer.index_1 > img {
			left: 20px;
			top: 30px;
			}
	.w-parallaximus-layer.index_2 {
		width: 500px;
		height: 600px;
		}
		.w-parallaximus-layer.index_2 > img {
			left: 40px;
			top: 50px;
			}

3. Initialize widget (usually at the end of the body section). For settings list, check options section

<script>
jQuery(document).ready(function($){
	$('.w-parallaximus').parallaximus({
		// options are located here
		// for example: set return-effect duration to 5 seconds
		duration: 5000
	});
});
</script>
<script>
window.addEvent('domready', function(){
	new Parallaximus(document.getElement('.w-parallaximus'), {
		// options are located here
		// for example: set return-effect duration to 5 seconds
		duration: 5000
	});
});
</script>

4. Now have a look at your widget. It probably needs some finishing touch (in your page styles CSS file):

  • set the correct width and height for your widget: 
    .w-parallaximus {
	width: 240px;
	height: 400px;
	}
  • do the same for each Parallaximus layer:

    Class name index_1 is connected with the farthest layer (HTML marked with the same class name). Next layer will be marked with index_2. It is not nessessary but used for easier style definitions.

    Layers usually should be bigger than the widget itself. How to choose layer width and height? The main rule: the closer the layer to viewpoint, the bigger it should be.

    .w-parallaximus-layer.index_1 {
	width: 240px;
	height: 400px;
	}
  • put layer images into their correct positions:

    It you didn't change layer sizes you can use layer masks in demo PSD file for easier measurement.

    .w-parallaximus-layer.index_1 > img {
	left: 20px;
	top: 30px;
	}

Your widget is ready!

Advanced

Responsive behaviour

Choose the way you widget resizes by adding or removing container classes width_fixed and height_fixed

Fixed width and height (example):

<div class="w-parallaximus width_fixed height_fixed">

Responsive width with fixed height (example):

<div class="w-parallaximus height_fixed">

Responsive width and height (example):

<div class="w-parallaximus">

Auto-init

If you don't want to manually initialize widget, you can use auto initialization. Simply add i-autoinit class to widget container:

<div class="w-parallaximus i-autoinit">

You can set widget options as shown below:

<div class="w-parallaximus i-autoinit" onclick="return {duration: 5000 /* options */}">

Options

Variable Default value Description
angleXRange 10 Range of horisontal rotation for the nearest layer (more distant layers are rotated at a smaller angle).
angleYRange 10 Range of vertical rotation for the nearest layer (more distant layers are rotated at a smaller angle).
basePoint [.5, .5] Point for basic position (when not controlled by mouse cursor). Can vary from [0, 0] to [1, 1]. [.5, .5] is the center of the widget.
duration 2000 Returning-to-basepoint duration (in ms: 2000 = 2 seconds)
fps 60 Frame per second limit for rendering (to reduce processor load for too frequent renders)
link false Link for widget banner (when filled, parallaximus becomes clickable)
perspective 400 Perspective of 3d effects
resizeDelay 50 Resize delay to reduce resize event calls
transition
'easeOutElastic'
Fx.Transitions.Elastic.easeOut
Returning-to-basepoint easing function (read more about easings)
Returning-to-basepoint transition function (read more about transitions)
use3d true Enable 3d transformations

API

Obtain Parallaximus object:

var parallaximus = $('.w-parallaximus').data('parallaximus');
var parallaximus = document.getElement('.w-parallaximus').retrieve('parallaximus');

Public methods:

// Quickly set position to certain point:
// [x, y] - both coords are limited by range from 0 to 1
parallaximus.set([.1, .1]);

// Animate position to certain point:
// [x, y] - both coords are limited by range from 0 to 1
parallaximus.start([.1, .1]);

// Cancel animation
parallaximus.cancel();

// Update widget after manual container resize
parallaximus.handleResize();
